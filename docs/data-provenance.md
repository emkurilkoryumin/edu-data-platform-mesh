# Происхождение данных

Проект использует воспроизводимые синтетические данные учебного стенда, а не реальные
персональные данные юных художников и не внешний публичный датасет.

## Источники внутри проекта

- `src/edu_platform/etl/gallery_sources.py` - единственная точка генерации demo-данных.
- `simulate_artwork_submissions()` имитирует ответ старого API галереи
  `api://old-gallery/submissions`.
- `build_artist_profiles()` и `write_artist_profiles_csv()` имитируют CSV-выгрузку
  административной панели старого сайта.
- `src/edu_platform/etl/raw_loader.py` записывает эти данные в Raw-слой и создаёт lineage
  events с входами `api://old-gallery/submissions` и `file://...artist_profiles_<date>.csv`.

## Почему данные не взяты "из воздуха"

Данные не вводятся руками в интерфейсе и не подставляются произвольным текстом. Они
строятся кодом по фиксированным правилам:

- генерация детерминирована seed-значениями и датой запуска;
- идентификаторы работ и заявок формируются программно;
- возраст художников ограничен диапазоном 6-17;
- техники, категории, регионы и типы наставников берутся из явных справочников в коде;
- `moderation_score` находится в диапазоне 0-100;
- для заявок сохраняется `source_system = "gallery_submissions_api"`.

Это делает набор данных повторяемым и проверяемым для защиты учебного проекта.

## Контроль качества

Правила качества лежат в `great_expectations/expectations/` и проверяются тестами:

- непустые ключи;
- уникальность `submission_id` и `artist_id`;
- допустимый возраст художника;
- допустимый диапазон `moderation_score`;
- допустимые значения техник и возрастных групп.

Локальный lightweight-runner `src/edu_platform/quality/validate_raw.py` использует эти
expectation suites как источник правды для unit-тестов и DAG-загрузки.

## Как подключить реальные данные

Для реального внедрения нужно заменить генераторы в `gallery_sources.py` на коннекторы к
настоящему API старого сайта и реальной CSV-выгрузке админки, сохранив те же схемы,
lineage inputs и quality suites. Остальная часть pipeline рассчитана на эти контракты.
